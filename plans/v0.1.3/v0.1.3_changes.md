# v0.1.3 Changes

## Summary

Added `dart_define_from_file` display support to the interactive shell UI. Users can now see which file is being used for dart defines when running `info -v` in the shell.

---

## Code Changes

### Modified Files

#### 1. `lib/src/core/app_context.dart`
- Added `dartDefineFromFile` getter to expose the dart define from file path from `BuildConfig`
- Follows existing pattern of convenience getters for verbose info display

#### 2. `lib/src/ui/shell.dart`
- Updated `_showInfo` method to display `dart_define_from_file` in verbose mode
- Shows "From File" line before individual dart define key-value pairs
- Only displays when `dartDefineFromFile` is configured

#### 3. `pubspec.yaml`
- Bumped version from 0.1.2 to 0.1.3

#### 4. `lib/src/version.dart`
- Updated `appVersion` constant from '0.1.2' to '0.1.3'

---

## User-Facing Changes

### Shell UI Enhancement
When running `flc` (interactive shell) and typing `info -v`:
- If `dart_define_from_file` is configured in `fluttercraft.yaml` (global or flavor-specific)
- The "Dart Define" section now shows "From File: <path>" as the first line
- Helps users understand where dart define values are sourced from

**Before:**
```
═══════════════════════════════════════
Dart Define
═══════════════════════════════════════
API_KEY             abc123
ENV                 production
```

**After:**
```
═══════════════════════════════════════
Dart Define
═══════════════════════════════════════
From File           .env.prod
API_KEY             abc123
ENV                 production
```
