# v0.1.4 Changes

## Summary
Fixed the incorrect `should_add_dart_define` flow.

## Breaking Changes

### Flag Renamed
- `should_add_dart_define` → `should_prompt_dart_define`
- Update your `fluttercraft.yaml` files

### Behavior Change
- **Before**: `global_dart_define` + `dart_define` only applied when flag was `true`
- **After**: Config-defined dart-defines **always** apply to builds
- Flag now only controls interactive prompting for custom dart-defines

## New Features

### Interactive Dart Define Input
When `should_prompt_dart_define: true`:
```
dart-define> MY_KEY=my_value
✓ Added: --dart-define=MY_KEY=my_value
dart-define>
```
